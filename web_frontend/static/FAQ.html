これは何？

@yak_ex による実験的サイトです。ソースを投げることで複数の C++ コンパイラでの実行結果が得られます。

無料？

とりあえず Amazon Web Services 無料利用枠(+$0.80/月(EBS)+α(S3))で運用されています。無料利用枠終了後にどうするか、というかそれまでに運用し続けるのかも含めて未定です。

内容を保存して後から参照できないの？

できるようになりました。http://ccf.myhome.cx:5000/result/0 のような形式で参照が可能です。また、http://ccf.myhome.cx:5000/results/ から過去の結果を辿ることが出来ます。

オープンソース？

はい。自分の書いた部分については Boost Software License, Version 1.0 です。The Chromium Project の Chromium から一部ソースを改変して使用していますが、その部分はオリジナルと同じライセンスになります。ソースコードについては Github に置いてあります。

○○コンパイラはないの？

とりあえず無償使用可能として公開されてるものが対象になります。VC7 (.NET 2003) は使用している sandbox ライブラリがそのままではコンパイルできないのでとりあえずパス。clang 3.1/3.2 は https://launchpad.net/~h-rayflood/+archive/llvm を、いくつかの gcc は https://launchpad.net/~ubuntu-toolchain-r/+archive/test を使用させて頂いています。他、Ubuntu 12.04 でバージョン共存可能なパッケージがあれば早いと思います。

Boost は？

ライブラリのビルドをしていないのでヘッダオンリーであれば 1.50 が使用可能です。1.50 なのは Ubuntu のパッケージに合わせただけなのでファイルを差し替えれば更新は可能です。反響次第でしょうか。

なんでポート 5000 なの？

Plack のデフォルトのままです。まぁ 80 番よりはアタック少ないかも、というのもあります。

CCF: ... とかいうメッセージが表示される

コンパイル/実行中のエラー/制限超過です。とりあえず、かなりぎりぎりの制限設定にしてあるためちょっと混んでたり複雑だったりするとすぐに制限超過に引っかかるかもしれません。その辺は今後調整できればと思っています。CCF を含むメッセージとしては以下のものがあります。

  • 終了ステータス
    基本的にエラーではなくコンパイル結果の実行時に 0 以外の終了ステータスが返ってきた場合です。上位 8bit が戻り値、下位 8bit がシグナルの情報になります。意図的に 0 以外のステータスを返せばどのフローで終了したか判別できます。
    • CCF: exit with non-zero status: 0xXXXX
  • 実行時エラー
    コンパイラの実行等でのエラーです。上位 8bit が戻り値、下位 8bit がシグナルの情報になります。
    • CCF: compilation failed by status: 0xXXXX
    • CCF: Adjujstment of obj prior to link failed by status: 0xXXXX
    • CCF: execution failed by status: 0xXXXX
    • CCF: link failed by status: 0xXXXX
  • サイズ制限
    現状、ソース、出力とも 10KiB 制限としています。
    • CCF: Source size is over 10KiB
    • CCF: Output size is over 10KiB
  • その他リソース制限
    実時間、メモリ、CPU時間で制限があります。
    • CCF: Time(real) limit exceeded
    • CCF: Memory limit exceeded
    • CCF: Time limit exceeded
  • システムコール制限
    Linux 上だと使用可能なシステムコールが制限されています。ここも厳しめなので要調整かとは思っています。なお、Windows 上では制限が無いわけではなくてメッセージ無しで失敗し処理が継続します。
    • CCF: Disallowed system call #...
