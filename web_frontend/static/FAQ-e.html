What is this?

This is an experimental site by @yak_ex. Posting a C++ source code, you can get results by several C++ compilers.

No fee?

At least, right now, it has been operated by Amazon Web Services Free Usage Tier(+$0.80/month(EBS)+alpha(S3)) but free tier is ended at July 2013. For now it continues but it has not been decided yet whether this site continues in future.

Can I refer the result afterwards?

Now, you can. You can refer the result by URL like http://ccf.myhome.cx:5000/result/0. Also, http://ccf.myhome.cx:5000/results/ shows previous results.

Is it open source?

Yes. What I wrote is distributed under the terms of Boost Software License, Version 1.0. Some codes are modified version of parts of Chromium by The Chromium Project. Their licence are the same as original. All source codes are available at Github.

Is there XYZ compiler?

If you want new compiler in this site, it should be available for free usage at least. For VC7 (.NET 2003), sandbox library is not compilable. Thus, it is not available in this site. clang 3.[0-4] are installed from https://launchpad.net/~h-rayflood/+archive/llvm, clang 3.[56] are installed from https://launchpad.net/~h-rayflood/+archive/llvm-upper, g++ 4.[789] are installed from https://launchpad.net/~h-rayflood/+archive/gcc-upper, and g++ 4.6 and 5 are installed from https://launchpad.net/~ubuntu-toolchain-r/+archive/test. If there is a package available for Ubuntu 12.04, it will be added soon, if you request.

Is Boost available?

1.55 is usable but libraries are not built.

Why is port 5000 used?

It is default port by Plack. It is considered that it may be less attacked than port 80.

The message "CCF: ..." is shown.

It is compiler error, runtime error or excess of limitation. Due to severe limitation, it may be easy to exceed the limitation if this site is a bit crowded or the source code is a bit complex. It should be adjusted gradually. Messages including CCF are as follows:

  • Exit status
    This is not an error but an exit status of compiled executable except for 0. Upper 8 bits represents the actual exit status and lower 8 bits represents signal. Returning the value except for 0 intentionally, you can recognize runtime flow.
    • CCF: exit with non-zero status: 0xXXXX
  • Runtime error
    Runtime error of compilation and others. Upper 8 bits represents the actual exit status and lower 8 bits represents signal.
    • CCF: compilation failed by status: 0xXXXX
    • CCF: Adjujstment of obj prior to link failed by status: 0xXXXX
    • CCF: execution failed by status: 0xXXXX
    • CCF: link failed by status: 0xXXXX
  • Size limitation
    Currently, both of source code and output are limited in 10KiB.
    • CCF: Source size is over 10KiB
    • CCF: Output size is over 10KiB
  • Resource limitation
    Actual runtime, memory, CPU time are limited.
    • CCF: Time(real) limit exceeded
    • CCF: Memory limit exceeded
    • CCF: Time limit exceeded
  • Syscall limitation
    Available system calls are limited on Linux. It might be necessary to adjust. On Windows, there IS limitation but no messages are produced and the execution continues.
    • CCF: Disallowed system call #...
